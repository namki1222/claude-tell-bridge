## 한 줄 요약

**loomo** = 여러 Claude Code / Codex 세션을 tmux 위에서 서로 대화하게 엮어주는(weave) 경량 CLI.
사람이 세션 사이를 복붙으로 중계하던 걸 없애고, **에이전트끼리 직접 메시지를 주고받게** 한다.
npm `@namki1222/loomo`, 명령어 `loomo`(+`tell` 별칭), GitHub `namki1222/loomo`.

---

## 왜 만들게 됐나 (motivation)

우리는 tmux로 **멀티 프로젝트 개발 플릿**을 운영한다 — 하울팟·팜페이·k팜·페보 등, 각 프로젝트가 하나의 세션이고 그 안에 역할 패널(서버·플러터·대시보드)이 나뉘어 있다. 각 패널엔 Claude Code 세션이 떠 있다.

문제: **여러 세션을 조율하려면 사람이 손으로 중계**해야 했다. 서버 세션의 답을 복사해서 플러터 세션에 붙여넣고, 다시 그 답을 관리 세션에 옮기고… 세션이 늘수록 사람이 병목이 됐다.

착상: **세션끼리 직접 말하게 하자.** 한 세션이 다른 세션의 채팅 입력창에 tmux로 메시지를 꽂아 넣고, 상관관계 KEY로 응답을 매칭한다. 여기에 CLAUDE.md 규약을 얹어 에이전트가 "받으면 KEY로 답한다"를 스스로 알게 한다. — 이게 2026-07-02 v0.1.0 `tell-tmux`의 시작이다.

**베팅**: Claude/Codex가 세션 간 협업을 네이티브로 내놓기 전에, 이 방식이 표준이 될 거라는 것. 무겁게(데몬·자체 멀티플렉서) 가지 않고 **가벼움을 해자**로 삼는다. 포트폴리오 관점의 진짜 화폐는 다운로드 수가 아니라 GitHub 스타 + 화제성.

## 개발 히스토리 — "하다 보니 필요해서 만들었다"

### Day 1 (2026-07-02): 아이디어 → 라이프사이클 완성까지 하루에

**v0.1.0 — 핵심 메시징이 됐다.**
`tell`/`ws`/`init`/`adopt`/`doctor` + CLAUDE.md 규약 템플릿. tmux 세션 브릿지의 원형.
→ 바로 실전 버그들이 나옴: split-window 타깃을 세션명으로 줘서 패널이 엉킴(패널 id로 수정), 루프 안 tmux가 stdin(설정 라인)을 삼켜 다음 패널을 건너뜀(`</dev/null`로 방지), grep 이식성.

**포지셔닝을 잡다.**
README를 "세션끼리 대화하게 해주는 툴"로 재정렬. `tell` 메시징 문법은 앞에서 빼고 부록으로. **사람은 관리 명령만, 메시징은 에이전트가 쓴다**는 원칙 확립. 패키지명을 `claude-tell-bridge`로 통일.

**v0.2.0 — 온보딩이 불편해서 재설계.**
`hub`/`list`/`rm` 신설. 출력에서 내부 동작(CLAUDE.md·프롬프트 감지) 언급을 전부 숨김 — 쓰는 사람은 tmux 내부를 몰라도 되게. 마법사 중단(Ctrl-C) 시에도 입력분 저장하도록 INT trap. 허브는 단일 패널(이름=역할).

**v0.3.0 — 기존 대화를 이어야 했다.**
workspaces.conf 4번째 필드=대화 세션ID → `claude --resume`로 이어받기. `adopt`는 세션ID만 주면 디렉터리를 로그(cwd)에서 자동 감지. 허브는 포인터 파일로 싱글턴 강제. `tell up`, 탭 자동완성, `up --tabs`(macOS 터미널 탭 자동 오픈, iTerm2=탭 / Terminal.app=Cmd+T, 권한 없으면 새 창 폴백).

**v0.4.0 — 라이프사이클을 닫다.**
`rm`은 설정 + CLAUDE.md 규약 블록까지 제거(프로젝트 파일은 무손상). `tell down`으로 끄기. up↔down 대칭. 규약 템플릿에 "**되물음도 tell -r로**"(처리 중 확인 필요하면 발신자에게 tell로 물어라) 규칙 추가 → 곧 결함 발견: 하네스 권한승인은 사용자만 풀 수 있는데 발신자에게 물으면 루프. → "권한승인은 사용자에게, 대기 상태만 발신자에 통지"로 수정.

### Day 2 (2026-07-06): 다국어 → codex → loomo 리브랜딩 → 첫경험 다듬기

**v0.5.0 — 글로벌을 향해.** README를 en/ko/zh-CN 3개 언어로 분리.

**v0.6.0 — 패널 정리.** `exec claude`로 셸을 대체(에이전트 종료 시 패널 자동 정리). `tell layout`으로 tmux.conf 편집 없이 배치 변경.

**v1.0.0 — 가장 큰 전환: codex 지원 + loomo 리브랜딩.**
- 에이전트 프로필 추상화(`load_agent_profile`): cmd·규약파일(CLAUDE.md↔AGENTS.md)·프롬프트(❯↔›)·세션경로·resume를 에이전트별로. → **loomo가 Claude Code와 Codex를 둘 다 구동**하고, 둘이 **모델을 넘나들며 서로 메시지**를 주고받음(Claude 허브가 Codex 프로젝트에 명령).
- 패널별 에이전트(conf 5번째 필드) → 한 화면에 claude·codex 혼합.
- **claude↔codex 실제 핑퐁 검증 성공** — codex가 AGENTS.md 규약만으로(명시적 지시 없이) `tell -r` 응답. "규약만으로 동작"이 핵심 요구였는데 통과.
- codex 3버그 수정: KEY 잘림(Space+C-u prime), 회색 placeholder를 미제출로 오인(codex는 감지 스킵), sandbox가 tmux 조회 차단(`--sandbox danger-full-access`).
- 브랜드를 **loomo**(엮다/weave)로. "Weave your Claude Code & Codex sessions into a team that talks to each other."

**v1.1.0 — 영어화 + i18n 안전장치.**
CLI 전면 영어(글로벌 유입). 프로토콜 헤더 기본 영어 `[session request/reply - KEY from ...]`. 단 **`LOOMO_LANG=ko`면 한국어 헤더·템플릿 유지**(LANG=ko* 자동감지) → 기존 한국어 플릿 무손상.

**v1.2.0 (오늘) — 첫 경험 UX + 안전.**
온보딩을 직접 도그푸딩하며 마찰이 드러남 → 개선:
- **화살표 선택 전면화**: init/adopt/rm/down 모든 선택을 ↑↓ 피커로 통일(`pick_menu`/`choose`). 비TTY(에이전트·스크립트·파이프)는 기존 정적 프롬프트 + 안전 기본값 유지 → 자동화 안 깨짐.
- init을 단일 프로젝트 단위로, 만든 것만 즉시 실행, 세션·역할 이름 검증(공백/중복 금지 — "비서" 세션 자기 자신과 충돌 방지).
- **세션 캐스케이드 종료**: 패널 하나 닫으면(에이전트 종료/Ctrl-C) 프로젝트 세션 통째 종료. loomo가 만든 세션에만 tmux 훅(pane-exited/pane-died + remain-on-exit), 전역 tmux 설정 무손상. → 반쯤 살아있는 팀을 남기지 않게.

## 핵심 설계 결정 & 근거

- **가벼움이 해자.** 데몬 없음, 자체 멀티플렉서 없음, 단일 bash 바이너리 + 규약 템플릿. GUI·자체 TUI로 안 감.
- **사람 ≠ 에이전트 인터페이스.** 사람은 관리 명령(init/up/down/rm)+자연어만. 메시징(`tell`/`tell -r`)은 에이전트 전용. 문서에서 tmux 노출 최소화(세션=프로젝트 팀).
- **규약으로 동작.** 명령에 "tell로 응답해줘"를 안 써도, CLAUDE.md/AGENTS.md 규약만으로 에이전트가 KEY로 응답. 이게 검증의 성패 기준이었음.
- **하위호환 집착.** 설정 경로(`~/.config/claude-tell-bridge`)·규약 마커·`tell` 별칭·`LOOMO_LANG=ko` 전부 유지 → 실운영 플릿이 안 깨지게.
- **네이밍 3분리.** npm=`@namki1222/loomo`(unscoped `loomo`는 `boom` 유사명으로 거부됨) · GitHub=`namki1222/loomo` · 브랜드/명령어=`loomo`.
- **포트폴리오 화폐 = 스타·화제성**, 다운로드 아님. 리브랜딩으로 잃은 다운로드는 대부분 출시일 봇 스파이크라 실손실 미미 → 리브랜딩 적기라 판단.
